# Spring vs Spring Boot: 무엇이 다를까?

2단계의 첫 번째 주제, 'Spring과 Spring Boot의 차이점'에 오신 것을 환영합니다. 이 둘의 관계를 이해하면 왜 우리가 Spring Boot로 백엔드 개발을 시작하는지 명확히 알 수 있습니다.

가장 쉽게 비유하자면, **Spring**이 수많은 부품(나사, 너트, 엔진, 바퀴 등)을 각각 따로 구매해서 조립해야 하는 **'자동차 조립 키트'**라면, **Spring Boot**는 운전에 필요한 모든 것이 갖춰져 바로 운전만 하면 되는 **'완성된 자동차'**와 같습니다.

이제 기술적인 관점에서 자세히 살펴보겠습니다.

---

### 1. 스프링 프레임워크 (Spring Framework)

스프링 프레임워크는 자바(Java)를 이용한 엔터프라이즈급 애플리케이션을 개발할 수 있도록 방대한 기능들을 제공하는 거대한 **'기반(Framework)'**입니다. 여기서 '엔터프라이즈급'이란 대규모 데이터와 수많은 사용자의 요청을 처리해야 하는 기업 환경 수준의 크고 복잡한 시스템을 의미합니다.

**주요 특징:**

*   **강력하고 유연함:** 웹 개발, 데이터베이스 연동, 보안, 테스트 등 개발에 필요한 거의 모든 기능을 '모듈' 형태로 제공하여 개발자가 필요한 것을 선택해서 사용할 수 있습니다.
*   **설정의 복잡함:** 강력한 만큼, 개발자가 직접 설정해야 할 부분이 많습니다. 과거에는 어떤 라이브러리를 사용할지, 각 라이브러리의 버전은 어떻게 할지, 서로 어떻게 연결할지 등을 XML 파일이나 자바 코드로 하나하나 명시해야 했습니다. 이 과정이 매우 복잡하고 시간이 오래 걸렸습니다.
*   **핵심 철학 (IoC/DI, AOP):** 스프링의 가장 중요한 핵심입니다. (이후 주제에서 자세히 다룹니다.)
    *   **IoC (Inversion of Control, 제어의 역전):** 객체를 생성하고 관리하는 주도권이 개발자가 아닌 프레임워크(스프링 컨테이너)에게 넘어가는 것을 말합니다.
    *   **DI (Dependency Injection, 의존성 주입):** 필요한 객체를 외부(스프링 컨테이너)에서 주입해주는 방식입니다. 객체 간의 결합도를 낮춰 코드를 유연하게 만듭니다.

간단히 말해, **스프링은 기능은 막강하지만, 그 기능을 제대로 사용하기 위해선 배워야 할 것과 설정해야 할 것이 많은 전문가용 도구**에 가깝습니다.

### 2. 스프링 부트 (Spring Boot)

스프링 부트는 위에서 설명한 복잡한 스프링 프레임워크를 **훨씬 더 쉽고 빠르게 사용할 수 있도록 도와주는 도구**입니다. 스프링을 기반으로 하지만, 복잡한 초기 설정을 대부분 자동화하여 개발자가 비즈니스 로직(실제 기능 구현)에만 집중할 수 있게 해줍니다.

스프링 부트가 해결해주는 핵심 문제들은 다음과 같습니다.

*   **자동 설정 (Auto-Configuration):** 스프링 부트의 가장 강력한 기능입니다. 프로젝트에 추가된 라이브러리(의존성)를 보고 '아, 사용자가 웹 개발을 하려는구나!'라고 스스로 판단해서 필요한 설정들을 자동으로 구성해줍니다. 개발자가 수동으로 하던 복잡한 설정 과정이 획기적으로 줄어듭니다.
*   **스타터 의존성 (Starter Dependencies):** 웹 개발에 필요한 라이브러리 묶음(`spring-boot-starter-web`), 데이터베이스 연동에 필요한 묶음(`spring-boot-starter-data-jpa`)처럼, 특정 기능에 필요한 라이브러리들을 **'스타터(starter)'**라는 패키지로 제공합니다. 개발자는 스타터 하나만 추가하면 관련된 수많은 라이브러리들이 호환되는 버전으로 자동 추가되므로 의존성 관리가 매우 편해집니다.
*   **내장 웹 서버 (Embedded Web Server):** 톰캣(Tomcat) 같은 웹 서버가 내장되어 있습니다. 과거에는 웹 애플리케이션을 실행하기 위해 별도로 웹 서버를 설치하고 배포해야 했지만, 스프링 부트는 애플리케이션을 실행하면 내장 서버가 함께 구동되어 즉시 웹 서비스를 시작할 수 있습니다.

결론적으로 **스프링 부트는 '설정보다 관례(Convention over Configuration)'라는 철학을 바탕으로, 복잡한 설정을 대신해주어 빠르고 간편하게 스프링 애플리케이션을 만들 수 있게 해주는 도구**입니다.

### 요약 정리

| 구분 | 스프링 프레임워크 (Spring Framework) | 스프링 부트 (Spring Boot) |
| :--- | :--- | :--- |
| **개념** | 자바 애플리케이션 개발을 위한 포괄적인 프레임워크 | 스프링을 더 쉽고 빠르게 사용하기 위한 도구 |
| **설정** | 개발자가 직접 XML이나 Java 코드로 상세하게 설정 | 자동 설정을 통해 대부분의 설정을 자동으로 처리 |
| **의존성 관리** | 필요한 라이브러리와 버전을 개발자가 직접 관리 | 'starter'를 통해 관련 라이브러리들을 자동으로 관리 |
| **웹 서버** | 별도의 웹 서버(Tomcat 등)를 설치하고 설정해야 함 | 웹 서버가 내장되어 있어 바로 실행 가능 |
| **개발 속도** | 초기 설정이 복잡하여 개발 속도가 상대적으로 느림 | 설정이 간소화되어 프로토타입 및 애플리케이션 개발이 매우 빠름 |

**결론:**
백엔드에 처음 입문하신다면, **스프링 부트(Spring Boot)로 시작하는 것이 정답에 가깝습니다.** 스프링 부트를 사용하면서 스프링의 핵심 개념(IoC/DI 등)을 차근차근 학습하는 것이 훨씬 효율적입니다.
