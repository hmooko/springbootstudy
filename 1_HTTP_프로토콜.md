# 1단계: 백엔드 기초 - HTTP 프로토콜

Spring Boot 학습 여정의 첫 번째 단계인 **백엔드 기초 다지기**를 시작하겠습니다. 그중에서도 가장 기본이 되는 **HTTP 프로토콜**에 대해 알아보겠습니다.

### **1. HTTP 프로토콜이란?**

HTTP(HyperText Transfer Protocol)는 웹에서 클라이언트(Client)와 서버(Server)가 서로 데이터를 주고받기 위해 사용하는 통신 규칙(Protocol)입니다. 쉽게 말해, 여러분의 웹 브라우저(클라이언트)가 네이버 서버(서버)에게 "웹툰 페이지를 보여줘!"라고 요청하고, 네이버 서버가 "알겠어, 여기 웹툰 페이지야."라고 응답할 때 사용하는 언어라고 생각할 수 있습니다.

#### **1) 클라이언트-서버 모델 (Client-Server Model)**

HTTP 통신은 항상 클라이언트의 **요청(Request)**으로 시작되고, 서버의 **응답(Response)**으로 끝납니다.

*   **클라이언트(Client):** 서비스를 요청하는 쪽입니다. 주로 웹 브라우저, 스마트폰 앱 등이 해당됩니다.
*   **서버(Server):** 요청을 받아 처리하고, 결과를 응답해주는 쪽입니다. 웹 사이트의 데이터를 저장하고 관리하는 컴퓨터가 해당됩니다.

이처럼 요청하는 쪽과 응답하는 쪽이 명확하게 구분된 구조를 **클라이언트-서버 모델**이라고 부릅니다.

#### **2) HTTP 메서드 (HTTP Methods)**

클라이언트는 서버에게 요청할 때, "어무엇을 하고 싶은지"를 명확하게 알려줘야 합니다. 이때 사용하는 것이 **HTTP 메서드**입니다. 여러 메서드가 있지만, 가장 중요한 4가지를 먼저 알아보겠습니다. (이는 앞으로 만들 RESTful API의 핵심인 CRUD(Create, Read, Update, Delete) 작업과 직접적으로 연결됩니다.)

*   **GET:** **데이터 조회(Read)**
    *   서버로부터 특정 데이터를 가져오고 싶을 때 사용합니다.
    *   예: 웹 브라우저에서 `www.google.com`을 입력하면, 브라우저는 구글 서버에 GET 요청을 보내 메인 페이지를 받아옵니다. 블로그 게시글을 보거나, 상품 목록을 조회하는 경우가 해당됩니다.

*   **POST:** **데이터 생성(Create)**
    *   서버에 새로운 데이터를 만들고 싶을 때 사용합니다.
    *   예: 회원가입 양식을 채우고 '가입' 버튼을 누르거나, 블로그에 새 글을 작성하고 '저장' 버튼을 누를 때 POST 요청이 발생합니다.

*   **PUT:** **데이터 수정(Update)**
    *   서버에 있는 기존 데이터를 완전히 새로운 데이터로 덮어쓸 때 사용합니다.
    *   예: 사용자의 프로필 정보 전체(이름, 주소, 연락처 등)를 한 번에 수정할 때 사용될 수 있습니다.

*   **DELETE:** **데이터 삭제(Delete)**
    *   서버에 있는 기존 데이터를 삭제하고 싶을 때 사용합니다.
    *   예: 작성했던 게시글을 삭제하거나, 장바구니에 담았던 상품을 삭제하는 경우가 해당됩니다.

#### **3) HTTP 상태 코드 (HTTP Status Codes)**

서버는 클라이언트의 요청을 처리한 후, 그 **결과가 어땠는지**를 알려주는 세 자리 숫자를 응답에 포함합니다. 이를 **상태 코드**라고 합니다. 자주 보게 될 대표적인 상태 코드는 다음과 같습니다.

*   **2xx (성공):** 요청이 성공적으로 처리되었음
    *   `200 OK`: 요청이 성공적으로 완료됨 (주로 GET, PUT 요청에 대한 응답)
    *   `201 Created`: 요청이 성공하여 새로운 리소스가 생성됨 (주로 POST 요청에 대한 응답)

*   **4xx (클라이언트 오류):** 요청이 잘못되었음
    *   `400 Bad Request`: 요청의 문법이 잘못되었거나, 서버가 이해할 수 없는 요청일 때
    *   `404 Not Found`: 요청한 리소스(페이지, 파일 등)를 서버에서 찾을 수 없을 때 (가장 흔하게 보는 오류!)
    *   `403 Forbidden`: 클라이언트가 해당 리소스에 접근할 권한이 없을 때

*   **5xx (서버 오류):** 서버에 문제가 발생했음
    *   `500 Internal Server Error`: 서버 내부에서 예상치 못한 오류가 발생했을 때
