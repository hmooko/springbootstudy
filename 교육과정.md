### **1단계: 백엔드 기초 다지기 (Foundation)**

Spring Boot를 배우기 전에, 웹이 어떻게 동작하는지 이해하는 것이 중요합니다. 이 단계에서는 백엔드 개발에 필요한 최소한의 기본 지식을 쌓습니다.

- **학습 목표:** 웹 통신의 기본 원리를 이해하고, 간단한 API의 개념을 파악한다.
- **핵심 주제:**
    1. **HTTP 프로토콜:**
        - Client-Server 모델 이해하기
        - HTTP Method (GET, POST, PUT, DELETE)의 의미와 용도
        - HTTP Status Code (200, 404, 500 등)의 의미
    2. **API와 RESTful API:**
        - API(Application Programming Interface)란 무엇인가?
        - REST 아키텍처의 기본 원칙 (자원, URI, 표현 등)
    3. **빌드 도구 (Build Tools):**
        - **Maven** 또는 **Gradle**의 기본 개념과 사용법 익히기 (의존성 관리, 프로젝트 빌드)
        - Spring Boot는 주로 Gradle을 많이 사용하지만, Maven도 좋습니다. 하나를 선택해서 익숙해지세요.
    4. **JSON (JavaScript Object Notation):**
        - API를 통해 데이터를 주고받는 표준 형식인 JSON의 구조와 문법 학습하기

> 💡 미니 프로젝트 #1: 내가 좋아하는 영화나 책 목록을 설명하는 간단한 RESTful API 명세를 직접 설계하고 작성해보기 (코딩 없이, 문서로만!)

### **2단계: Spring Boot 첫걸음 (First Steps)**

이제 본격적으로 Spring Boot를 시작합니다. 프레임워크의 핵심 철학과 기본 구조를 이해하는 데 집중합니다.

- **학습 목표:** Spring Boot 프로젝트를 생성하고, 간단한 API를 만들어 실행할 수 있다.
- **핵심 주제:**
    1. **Spring과 Spring Boot의 차이점:** 왜 Spring Boot가 더 편리하고 강력한지 이해하기
    2. **IoC (Inversion of Control)와 DI (Dependency Injection):**
        - 스프링의 가장 핵심적인 개념! 객체를 직접 생성하는 것이 아니라 스프링 컨테이너가 대신 관리해주는 이유와 장점 이해하기
        - `@Component`, `@Service`, `@Repository`, `@Controller` 어노테이션의 역할
    3. **프로젝트 생성:**
        - **Spring Initializr** ([https://start.spring.io/](https://start.spring.io/))를 사용하여 프로젝트 생성하기
    4. **기본 API 만들기:**
        - `@RestController`, `@RequestMapping`, `@GetMapping`을 사용하여 "Hello, World!"를 반환하는 API 만들기
        - 다양한 요청 처리: `@RequestParam`, `@PathVariable`, `@RequestBody` 사용법

> 💡 미니 프로젝트 #2: 간단한 "오늘의 명언" API 만들기. /quote로 GET 요청을 보내면 랜덤 명언을 텍스트로 반환하는 기능을 구현해보세요. (데이터는 코드 안에 하드코딩)

### **3단계: 데이터베이스 연동 (Connecting to Database)**

애플리케이션의 핵심은 데이터입니다. 이제 데이터를 영구적으로 저장하고 관리하는 방법을 배웁니다.

- **학습 목표:** Spring Boot 애플리케이션을 데이터베이스와 연동하여 데이터를 저장, 조회, 수정, 삭제(CRUD)할 수 있다.
- **핵심 주제:**
    1. **데이터베이스 기초:**
        - 관계형 데이터베이스(RDBMS) 기본 개념 (테이블, 레코드, PK/FK 등)
        - 가볍게 시작하기 좋은 **H2** 또는 **SQLite** 데이터베이스, 실무에서 많이 쓰는 **MySQL** 또는 **PostgreSQL** 중 하나를 선택하여 설치 및 사용법 익히기
    2. **JPA (Java Persistence API)와 Hibernate:**
        - SQL 쿼리를 직접 작성하지 않고 Java 객체로 데이터베이스를 다루게 해주는 ORM(Object-Relational Mapping) 기술
        - `@Entity`, `@Id`, `@GeneratedValue` 등 주요 어노테이션
    3. **Spring Data JPA:**
        - JPA를 더 쉽게 사용하게 해주는 기술
        - `JpaRepository` 인터페이스를 상속받아 기본적인 CRUD 메서드를 자동으로 구현하기
        - *MVC 패턴 (Model-View-Controller)**을 적용하여 **Controller - Service - Repository** 계층 구조로 코드 분리하기

> 💡 미니 프로젝트 #3: 간단한 To-Do List API 만들기. 할 일(Todo)을 생성(POST), 전체 조회(GET), 특정 할 일 조회(GET), 완료 처리(PUT), 삭제(DELETE)하는 기능을 구현하세요. 이제 데이터는 H2 같은 인메모리 DB나 MySQL에 저장됩니다.

### **4단계: 실전 기능 추가 (Advanced Features)**

이제 기본적인 API 서버를 넘어, 실제 서비스에 필요한 고급 기능들을 추가해봅니다.

- **학습 목표:** 인증, 예외 처리, 데이터 유효성 검사 등 실무에 필수적인 기능들을 구현할 수 있다.
- **핵심 주제:**
    1. **Spring Security:**
        - 애플리케이션 보안의 핵심!
        - 기본적인 사용자 인증(로그인) 및 인가(권한 부여) 구현하기
        - *JWT (JSON Web Token)**를 이용한 토큰 기반 인증 방식 학습 (현대 API 서버의 표준)
    2. **예외 처리 (Exception Handling):**
        - `@ExceptionHandler` 와 `@ControllerAdvice`를 사용하여 애플리케이션 전역에서 발생하는 예외를 깔끔하게 처리하고 일관된 에러 응답 보내기
    3. **유효성 검사 (Validation):**
        - `spring-boot-starter-validation` 의존성을 추가하여 `@Valid`, `@NotEmpty` 등 어노테이션으로 클라이언트 요청 데이터가 유효한지 검증하기

> 💡 미니 프로젝트 #4: To-Do List API에 사용자 기능 추가하기. 회원가입, 로그인 API를 만들고, JWT를 발급합니다. 이제 각 사용자는 자신의 할 일 목록만 보고 수정할 수 있도록 API를 보안 처리하세요.

### **5단계: 배포와 심화 학습 (Deployment & Beyond)**

이제 내가 만든 애플리케이션을 다른 사람도 사용할 수 있도록 세상에 공개하는 방법을 배웁니다.

- **학습 목표:** 애플리케이션을 테스트하고, 컨테이너화하여 클라우드에 배포할 수 있다.
- **핵심 주제:**
    1. **테스팅 (Testing):**
        - JUnit5와 Mockito를 이용한 **단위 테스트(Unit Test)** 작성하기
        - `@SpringBootTest`를 이용한 **통합 테스트(Integration Test)** 작성하기
    2. **로깅 (Logging):**
        - SLF4J와 Logback을 이용해 애플리케이션 동작 기록을 남기는 방법
    3. **컨테이너화 (Containerization):**
        - **Docker**의 기본 개념을 배우고, 내 Spring Boot 애플리케이션을 Docker 이미지로 만들기
    4. **배포 (Deployment):**
        - AWS, Google Cloud Platform (GCP), Naver Cloud Platform 같은 클라우드 서비스의 무료 티어를 이용해 내 애플리케이션을 배포해보기
> 💡 미니 프로젝트 #4: 지금까지 배운 모든 것을 종합하여 To-Do List API 완성하기. 테스팅, 로깅 등을 포함하여 자신만의 프로젝트를 완성하고 Docker를 이용해 배포까지 도전해보세요!
